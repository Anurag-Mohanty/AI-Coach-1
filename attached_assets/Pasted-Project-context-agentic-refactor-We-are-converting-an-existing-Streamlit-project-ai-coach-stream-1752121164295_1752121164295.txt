Project context (agentic refactor)
We are converting an existing Streamlit project (“ai-coach-streamlit”) into a clean multi-agent system.
Two hard rules:

Only three JSON blobs serve as contracts between layers

user_current_capabilities – skills/tools/roles/confidence (schema in schemas/user_current_capabilities.schema.json)

user_target_role_profile – target role profile (schema exists)

user_gap_analysis – missing skills + priority (schema exists)

Tools are memory-free pure functions.

They never read/write global_agent_memory.

Orchestrators handle all memory IO via services/memory_store.save_blob / load_blob.

Tools accept explicit arguments for any upstream data they need and return a dict that validates against one of the three schemas.
